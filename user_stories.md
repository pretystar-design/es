# User Stories for 图形化 Terraform 代码生成工具

## 1. 基础资源编辑
- **作为** 新手运维人员，
- **我想要** 在画布上拖拽创建 AWS、Azure、GCP 等资源节点，
- **从而** 直观地构建基础设施而不需要手写 HCL。

## 2. 属性配置
- **作为** 开发者，
- **我想要** 选中资源后在属性面板中填写所有必填字段（如 AMI、instance_type），
- **从而** 确保资源配置完整且符合 Provider schema。

## 3. 实时 HCL 预览
- **作为** 基础设施工程师，
- **我想要** 每次画布改动后即时看到对应的 HCL 代码，
- **从而** 验证生成的代码是否符合预期并可直接保存。

## 4. 代码导出
- **作为** CI/CD 管理者，
- **我想要** 将当前画布生成的 HCL 一键导出为 `.tf` 文件或 ZIP 包，
- **从而** 能够快速把代码提交到代码库进行审查与部署。

## 5. Provider 与模块库
- **作为** 多云团队成员，
- **我想要** 在左侧资源库里搜索并直接拖入任何 Terraform Provider 官方资源，
- **从而** 不必手动查阅文档，提升建模速度。

## 6. 代码验证 & 计划
- **作为** 运维主管，
- **我想要** 点击“Validate/Plan”按钮让后台执行 `terraform validate` 与 `terraform plan`，
- **从而** 及时发现语法或依赖错误，并在画布节点上高亮显示。

## 7. 现有基础设施导入
- **作为** 已有项目维护者，
- **我想要** 导入已有的 `tfstate` 或项目文件，让工具自动在画布上生成对应节点，
- **从而** 能够在可视化界面继续迭代已有基础设施。

## 8. 插件化扩展
- **作为** 产品经理，
- **我想要** 通过插件 API 为公司内部自研 Provider 定义 UI 组件，
- **从而** 在同一工具内统一管理内部专有资源。

## 9. 项目切换与版本管理
- **作为** 团队成员，
- **我想要** 在 UI 中切换不同项目、保存画布模型、回滚历史版本，
- **从而** 支持多分支开发与审计追踪。

## 10. 权限与审计
- **作为** 安全合规负责人，
- **我想要** 系统使用企业 SSO 登录并记录所有操作日志，
- **从而** 满足合规审计需求并防止未授权修改。

## 11. 本地化与多语言
- **作为** 海外团队成员，
- **我想要** 切换 UI 语言（中文/英文），
- **从而** 使用熟悉的语言进行操作。

## 12. CI 集成
- **作为** DevOps 自动化负责人，
- **我想要** 在 CI 流水线中调用工具的 CLI/REST 接口生成 HCL 并执行 `terraform plan`，
- **从而** 将可视化设计同步到自动化部署流程。
